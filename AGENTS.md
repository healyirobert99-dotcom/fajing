# AGENTS.md

## 项目定位

本仓库保存长篇司法悬疑与东方民俗小说《法警·行刑之后》。

GitHub远端仓库是项目唯一正式版本。

## 角色分工

* 用户负责项目决策、司法警务真实性和最终定稿。
* ChatGPT负责策划、主笔、审稿、修订和连续性判断。
* Codex负责仓库读取、精确修改、差异检查、提交和推送。
* Codex不得自行担任小说主笔。

## 开始任务前

1. 检查当前分支和工作区状态。
2. 存在未提交修改时停止，不得覆盖或清理。
3. 工作区干净时执行 `git fetch origin` 和 `git pull --ff-only`。
4. 按需读取：

   * `DECISIONS.md`
   * `WRITING_STYLE.md`
   * 当前权威总纲
   * `PROJECT_STATUS.md`
   * 目标章节和相关章节

凡涉及章节规划、正文写作、正文重写、文学审稿、语言修订或连续性审查，`WRITING_STYLE.md` 不是"按需读取"，而是必须读取。

## 工作模式

### 只读审计

用户要求检查、搜索、比较、审计或确认时：

* 不修改文件；
* 不提交；
* 不推送；
* 只报告事实、冲突和影响。

### 正式入库

只有用户明确要求定稿入库、提交或推送时才能修改。

严格执行用户或ChatGPT提供的文件清单和最终文本。

## 写作风格强制约束

* 凡涉及章节规划、正文写作、正文重写、文学审稿、语言修订或连续性审查，开始前必须读取远端最新版本的 `WRITING_STYLE.md`。
* `WRITING_STYLE.md` 是本项目正文叙事表达、场景组织、对话、推理、描写、段落、节奏和语言审查的正式详细规范。
* 所有新正文、重写稿和文学修订稿必须按照 `WRITING_STYLE.md` 执行。
* 不得继续以历史章节的具体句法、段落长度、动作顺序或语言习惯替代 `WRITING_STYLE.md`。
* 第二章《快开枪》仍是正式正文和连续性依据，但不再单独承担全书详细写作风格规范的功能。
* `WRITING_STYLE.md` 只约束写作表达，不得覆盖用户最新明确决定、故事事实、人物知情范围、司法流程、权威总纲、章节连续性和已经锁定的结构。
* 写作风格范围内发生冲突时，优先级为：
  1. 用户在当前任务中的最新明确决定；
  2. `WRITING_STYLE.md`；
  3. `DECISIONS.md` 中此前的概括性风格要求；
  4. 已定稿章节的局部语言习惯；
  5. 历史参考稿。
* Codex不得根据 `WRITING_STYLE.md` 自行创作、润色或改写正文。Codex只负责读取文件、执行用户或ChatGPT提供的精确正文、检查diff、提交和推送。
* 如果用户或ChatGPT提供的入库文本与 `WRITING_STYLE.md` 存在明显冲突，Codex只报告冲突，不得擅自修改正文。

## 修改边界

未经明确授权，不得：

* 自行创作、润色或重写正文；
* 新增情节、设定、人物、组织或能力；
* 修改未列出的章节；
* 顺带修复其他问题；
* 调整人物知情范围和伏笔顺序；
* 修改总纲、决定或项目状态；
* 为AI检测结果擅自改写文本；
* 建立新的管理文件或规则。

发现其他问题只报告，不修改。

## 文件规则

* 保持UTF-8编码。
* 保持中文文件名和章节编号。
* 完整正文按交接稿原样写入。
* 局部替换前必须确认原文准确匹配。
* 如果仓库内容与交接指令不一致，停止并报告。
* 不将工作说明写入小说正文。

## Git规则

禁止：

* `git reset --hard`
* 强制推送
* 自动变基
* 改写历史
* 擅自删除分支
* 擅自丢弃本地修改

提交前执行：

* `git diff --check`
* `git diff --stat`
* 检查完整diff

只有任务明确要求时才能commit和push。

完成后返回：

* 修改文件列表；
* diff摘要；
* commit SHA；
* push结果；
* 最终工作区状态。

## 当前创作阶段

从远端最新状态开始：

1. 重新审稿第一至第八章；
2. 每章确认后分别入库；
3. 完成整体连续性检查；
4. 再规划和创作第九章。

Codex不得自行推进章节创作。
