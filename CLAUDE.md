# 《法警刑官》项目工作规则

## 1. 项目定位

本项目用于创作小说《法警刑官》。

当前工作方式为：

- WorkBuddy负责章节规划、正文初稿和文学性修改；
- Claude Code可用于文件管理、机械检查和定向修改，但不再作为正文主执笔；
- ChatGPT负责外部审稿、逻辑检查和修改意见；
- 用户负责批准故事决定和正文修改；
- 外部审稿意见只有经过用户确认后才能执行。

不得重新设计全书框架。

## 2. 每次启动后的固定流程

接到创作任务后，必须先执行：

1. 读取 `PROJECT_STATUS.md`；
2. 读取 `DECISIONS.md`；
3. 读取当前任务指定的案件框架或章节正文；
4. 检查 Git 工作区是否存在未提交修改；
5. 向用户简要报告当前阶段、当前唯一任务和禁止事项；
6. 用户没有明确要求写作时，不得直接修改正文。

如果存在远程 Git 仓库：

- 工作区干净时，可以执行 `git pull --ff-only`；
- 工作区有未提交内容时，不得自动拉取或覆盖，必须先报告。

## 3. 资料优先级

资料冲突时，按照以下优先级：

1. 用户最新明确确认的决定；
2. `DECISIONS.md`；
3. 当前案件正式框架；
4. 完整创作总纲；
5. 已经批准的正文；
6. 插件生成内容；
7. Agent自行提出的建议。

低优先级内容不得覆盖高优先级内容。

发现冲突时只报告，不得擅自选择新方案。

## 4. 创作边界

- 一次只执行 `PROJECT_STATUS.md` 中的当前唯一任务。
- 未经用户要求，不得继续下一章。
- 不得新增人物、能力、道具、组织、支线或世界观规则。
- 不得改变案件真相和人物核心动机。
- 不得改变已经确定的信息释放顺序。
- 不得把备选建议自动写入正式设定。
- 不得提前泄露当前人物尚不知道的信息。
- 不得使用通用网文模板覆盖现有框架。
- 不得为了制造爽点擅自增加反转、战斗或境界突破。
- 数值化能力只供作者内部参考，正文不直接显示数值。
- 正文原则上采用第三人称有限视角，跟随陈铮。
- 未经核实的司法、枪械、押解和卷宗细节必须标记为"待核实"。

未经用户明确批准，不得新增规则、扩展任务或改变研究口径。

## 5. 正文修改规则

修改已有章节时：

1. 优先局部修改，不得无理由整章重写；
2. 保留用户已经确认有效的内容；
3. 修改前说明准备调整的位置和原因；
4. 修改后列出实际修改内容；
5. 不得同时制造多个用途不明的正文版本；
6. 未经批准不得删除原稿。

## 6. webnovel-writer使用范围

允许使用：

- 章节规划；
- 单章初稿生成；
- 人物状态记录；
- 已公开信息记录；
- 连续性检查；
- OOC检查；
- 节奏检查。

禁止使用：

- 自动重构总纲；
- 自动连续生成多章；
- 自动增加角色和支线；
- 根据插件评分改变既定案件；
- 自动修订已经批准的设定；
- 使用插件默认世界观替换本项目世界观。

## 7. 每次任务完成后的固定流程

任务完成后必须：

1. 更新 `PROJECT_STATUS.md`；
2. 将用户正式确认的新决定写入 `DECISIONS.md`；
3. 列出本次修改的文件；
4. 记录尚未解决的问题；
5. 设置下一步唯一任务；
6. 未经用户要求，不得自动开始下一步。

如果项目已经配置 Git：

- 检查差异；
- 使用清楚的中文提交说明；
- 只有在能够确认远程仓库为用户项目且不会覆盖他人修改时才推送；
- 有冲突时停止并报告，不得强制覆盖。
